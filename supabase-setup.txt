Supabase Setup - A Seis Manos

1. Crea un proyecto en Supabase
   - Entra a https://app.supabase.com y crea un proyecto nuevo.
   - Selecciona la región más cercana y anota el nombre de tu proyecto.

2. Configura las credenciales del proyecto
   - Ve a Settings > API.
   - Copia la URL del proyecto (`https://<YOUR_PROJECT>.supabase.co`).
   - Copia la llave pública (`anon` key).

3. Crea las tablas y funciones
   - Abre el SQL Editor en Supabase.
   - Copia y pega el contenido del archivo `supabase-schema.sql`.
   - Ejecuta el script completo.

4. Crea el usuario administrador
   - En Authentication > Users, crea un usuario admin.
   - Usa el email que quieras, por ejemplo `admin@restaurante.com`.
   - Si Supabase requiere confirmación de email, confirma ese usuario.

5. Registra el admin en la tabla `public.app_admins`
   - En SQL Editor ejecuta:
     insert into public.app_admins (user_id, email)
     select id, email
     from auth.users
     where email = 'admin@restaurante.com'
     on conflict (user_id) do update
     set email = excluded.email;

6. Actualiza `supabase-config.js`
   - Abre `supabase-config.js` en el proyecto.
   - Reemplaza:
     - `https://YOUR_PROJECT.supabase.co` con tu URL real.
     - `YOUR_SUPABASE_ANON_KEY` con la `anon` key.

7. Verifica el flujo
   - El admin debe poder iniciar sesión con su correo y contraseña.
   - El admin puede asignar almuerzos pagos por cédula.
   - El cliente puede ingresar su número de identificación y ver su saldo.
   - El admin puede marcar almuerzos gratis como usados.

8. Opcional: Servir la app localmente
   - Si quieres probar localmente, usa un servidor estático simple.
   - Por ejemplo, con Python:
     python3 -m http.server 8000
   - Luego abre `http://localhost:8000` en el navegador.

Notas importantes
   - No uses la `service_role` key en el navegador ni en GitHub.
   - El flujo actual ya no usa códigos para el cliente. El admin asigna almuerzos y el cliente solo consulta su saldo.
